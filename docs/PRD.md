# 产品需求文档 (PRD) - 今天又活了一天

## 1. 产品概述
"今天又活了一天" 是一款以"生存确认"和"记忆沉淀"为核心概念的打卡工具类产品。它通过轻量化的每日互动，引导用户记录生活中的意义，并为重要的人建立情感档案。

### 1.1 产品定位
- **核心驱动**: 核心在于"活着"的自我感和"记忆"的构建
- **工具定位**: 产品自定位为"打卡工具"，但不强求过度记录，支持"什么都不记录"的纯粹打卡
- **情感连接**: 通过文字记录来对抗遗忘，将重要的人和事内化为持久的记忆

### 1.2 目标用户
- 希望养成每日反思习惯的用户
- 想要记录生活中有意义时刻的用户
- 希望建立人际关系档案、珍惜重要人物的用户

## 3. 功能模块详细说明

### 3.1 主页模块

主页作为应用的核心入口，聚合最重要的功能和信息。

#### 3.1.1 生存状态卡片
| 功能点 | 说明 |
|--------|------|
| 状态展示 | 显示当前打卡状态（待确认/已确认） |
| 心跳动画 | 生存状态的视觉化表达 |
| 核心文案 | "原来你还活着啊" |

#### 3.1.2 快速打卡按钮
| 功能点 | 说明 |
|--------|------|
| 打卡入口 | 一键确认存活，支持早晚打卡 |
| 反馈动画 | 成功打卡后展示"恭喜你又活过了一天" |
| 状态变更 | 打卡后按钮变为已完成状态 |

#### 3.1.3 今日生日提醒
| 功能点 | 说明 |
|--------|------|
| 提醒展示 | 仅在有人物生日当天显示 |
| 快捷入口 | 点击可跳转至对应人物详情 |
| 设计理念 | 不提前提醒，尊重"真正在乎会主动定闹钟"的理念 |

#### 3.1.4 每日信息差
| 功能点 | 说明 |
|--------|------|
| 内容类型 | 每日名言/小知识/生活感悟 |
| 展示形式 | 卡片式，渐变背景突出展示 |
| 更新频率 | 每日更新 |

#### 3.1.5 今日有意义的事（快捷入口）
| 功能点 | 说明 |
|--------|------|
| 快速记录 | 文本输入框，支持快速记录 |
| 图片上传 | 支持添加今日代表性照片 |
| 保存记忆 | 一键保存到打卡记录 |

#### 3.1.6 未打卡提示
| 功能点 | 说明 |
|--------|------|
| 温和引导 | "一定是因为今天过得很开心吧，才忘了打卡" |
| 设计理念 | 非压力式催促，正向引导 |

## 4. 交互设计规范

### 4.1 打卡反馈文案

| 场景 | 文案 |
|------|------|
| 待打卡状态 | "原来你还活着啊" |
| 打卡成功 | "恭喜你又活过了一天！" |
| 昨日未打卡 | "一定是因为昨天过得很开心吧，才忘了打卡" |
| 连续未打卡 | "好久不见，你还好吗？" |

### 4.2 生日提醒规则
- 不提前数日提醒
- 仅在生日当天的打卡流程中提示
- 在主页和人物模块同时展示

### 4.3 死亡确认邮件规则
- 用户可自定义触发天数（3/7/14/30天）
- 邮件发送给预设的紧急联系人
- 邮件内容为确认用户生存状态的提醒

## 5. 设计理念

### 5.1 核心价值观
1. **轻量化**: 不强求过度记录，支持"什么都不记录"的纯粹打卡
2. **正向引导**: 所有反馈文案采用温和、非压力式的表达
3. **情感连接**: 通过文字记录来对抗遗忘
4. **记忆深化**: 写下来的过程本身就是加深记忆的过程

### 5.2 设计风格
- 简洁现代的UI风格
- 柔和的配色方案（slate灰 + 点缀色）
- 圆角卡片设计
- 适当的留白与层次感
