# 产品需求文档 (PRD) - 还活着

## 1. 产品概述

"还活着" 是一款以"生存确认"和"记忆沉淀"为核心概念的打卡工具类产品。它通过轻量化的每日互动，引导用户记录生活中的意义，并为重要的人建立情感档案。

### 1.1 产品定位

| 维度 | 说明 |
|------|------|
| **核心驱动** | "活着"的自我感知 + "记忆"的主动构建 |
| **工具定位** | 打卡工具，但不强求过度记录，支持"什么都不记录"的纯粹打卡 |
| **情感连接** | 通过文字记录来对抗遗忘，将重要的人和事内化为持久的记忆 |

### 1.2 目标用户

- 希望养成每日反思习惯的用户
- 想要记录生活中有意义时刻的用户
- 希望建立人际关系档案、珍惜重要人物的用户

### 1.3 核心价值观

| 价值观 | 说明 |
|--------|------|
| **轻量化** | 不强求过度记录，支持"什么都不记录"的纯粹打卡 |
| **正向引导** | 所有反馈文案采用温和、非压力式的表达 |
| **情感连接** | 通过文字记录来对抗遗忘 |
| **记忆深化** | 写下来的过程本身就是加深记忆的过程 |

## 3. 功能模块

### 3.1 账户模块

#### 3.1.1 登录
| 功能点 | 说明 |
|--------|------|
| 邮箱登录 | 邮箱 + 密码登录 |
| 第三方登录 | 微信、Apple ID 快捷登录 |
| 记住登录 | 支持自动登录 |

#### 3.1.2 注册
| 功能点 | 说明 |
|--------|------|
| 邮箱注册 | 邮箱 + 密码 + 昵称（选填） |
| 密码强度 | 实时显示密码强度指示器（弱/中/强） |
| 密码要求 | 至少6位，包含字母和数字 |

#### 3.1.3 忘记密码
| 功能点 | 说明 |
|--------|------|
| 邮箱验证 | 发送验证码到注册邮箱 |
| 密码重置 | 验证通过后设置新密码 |
| 安全提示 | 重置成功后提醒其他设备重新登录 |

### 3.2 主页模块

主页作为应用的核心入口，聚合最重要的功能和信息。

#### 3.2.1 生存状态卡片
| 功能点 | 说明 |
|--------|------|
| 状态展示 | 显示当前打卡状态（待确认/已确认） |
| 心跳动画 | 生存状态的视觉化表达 |
| 核心文案 | "还活着吗？" |

#### 3.2.2 快速打卡按钮
| 功能点 | 说明 |
|--------|------|
| 打卡入口 | 一键确认存活 |
| 反馈动画 | 成功打卡后展示"恭喜你又活过了一天" |
| 状态变更 | 打卡后按钮变为已完成状态 |

#### 3.2.3 今日生日提醒
| 功能点 | 说明 |
|--------|------|
| 提醒展示 | 仅在有人物生日当天显示 |
| 快捷入口 | 点击可跳转至对应人物详情 |

#### 3.2.4 每日信息差
| 功能点 | 说明 |
|--------|------|
| 内容类型 | 每日名言/小知识/生活感悟 |
| 展示形式 | 卡片式，渐变背景突出展示 |
| 更新频率 | 每日更新 |

#### 3.2.5 今日有意义的事
| 功能点 | 说明 |
|--------|------|
| 快速记录 | 文本输入框，支持快速记录 |
| 图片上传 | 支持添加今日代表性照片 |
| 保存记忆 | 一键保存到打卡记录 |

### 3.3 打卡模块

打卡模块承载详细的记录功能和历史回顾。

#### 3.3.1 打卡统计
| 数据项 | 说明 |
|--------|------|
| 总打卡天数 | 累计打卡的总天数 |
| 连续打卡 | 当前连续打卡天数 |
| 记录条数 | 有意义事件的记录总数 |

#### 3.3.2 打卡日历
| 功能点 | 说明 |
|--------|------|
| 月度视图 | 日历形式展示当月打卡情况 |
| 状态标记 | 已打卡(绿色)、今天(边框)、未打卡(灰色)、补签(虚线) |
| 月份切换 | 支持前后月份切换查看 |

#### 3.3.3 近期记录列表
| 功能点 | 说明 |
|--------|------|
| 时间线展示 | 按日期倒序展示记录 |
| 记录内容 | 日期、打卡状态、意义记录文字 |
| 照片展示 | 展示当日上传的照片缩略图 |

#### 3.3.4 意义记录
| 功能点 | 说明 |
|--------|------|
| 文字记录 | 类似"摸鱼日报"的形式，记录有意义事件 |
| 今日照片 | 上传一张代表今天的照片 |
| 情绪标签 | (可选) 当日心情标签 |

#### 3.3.5 打卡补签
| 功能点 | 说明 |
|--------|------|
| 补签入口 | 点击日历中未打卡日期触发补签弹窗 |
| 补签限制 | 仅可补签过去7天内的记录 |
| 补签标记 | 补签日期显示特殊标记（虚线边框）以区分 |
| 补签次数 | 每月最多补签3次，防止滥用 |

#### 3.3.6 连续打卡里程碑
| 功能点 | 说明 |
|--------|------|
| 里程碑节点 | 7天、30天、100天、365天、1000天 |
| 动画反馈 | 达成里程碑时展示庆祝动画和鼓励文案 |
| 成就徽章 | 里程碑成就以徽章形式展示在个人页面 |
| 里程碑文案 | "连续7天，你是真的在认真活着" 等温暖文案 |

### 3.4 人物模块

人物模块是产品差异化的核心功能，用于建立情感档案。

#### 3.4.1 人物搜索与添加
| 功能点 | 说明 |
|--------|------|
| 搜索功能 | 按姓名/称呼搜索已添加人物 |
| 添加入口 | 顶部按钮 + 悬浮按钮双入口 |

#### 3.4.2 今日生日区域
| 功能点 | 说明 |
|--------|------|
| 突出展示 | 当日有生日的人物优先展示 |
| 送祝福入口 | 快捷入口进入详情页 |
| 年龄展示 | 显示当前年龄 |

#### 3.4.3 人物列表
| 功能点 | 说明 |
|--------|------|
| 卡片展示 | 头像、姓名、MBTI、生日、印象主题色 |
| 列表排序 | 按添加时间/生日临近排序 |
| 详情入口 | 点击进入人物详情页 |

#### 3.4.4 人物详情页
| 字段 | 说明 | 必填 |
|------|------|------|

#### 3.4.5 人物分组
| 功能点 | 说明 |
|--------|------|

| 自定义分组 | 支持用户创建自定义分组 |
| 分组筛选 | 按分组快速过滤人物列表 |
| 分组统计 | 各分组人数统计展示 |

#### 3.4.6 重要日期提醒
| 功能点 | 说明 |
|--------|------|
| 日期类型 | 生日、纪念日、自定义日期 |
| 多日期支持 | 每个人物可添加多个重要日期 |
| 日期备注 | 可为日期添加说明（如"相识纪念日"） |

## 4. 业务规则

### 4.1 打卡规则

| 规则 | 说明 |
|------|------|
| 打卡周期 | 每日0:00-23:59为一个打卡周期 |
| 打卡次数 | 每日仅可打卡一次 |
| 连续计算 | 连续天数从第一次打卡开始累计，中断后重新计算 |
| 补签规则 | 仅可补签过去7天内的记录，每月最多3次 |

### 4.2 提醒规则

| 规则 | 说明 |
|------|------|
| 打卡提醒 | 用户自定义时间，默认21:00 |
| 生日提醒 | 仅在生日当天提醒，不提前数日提醒 |
| 死亡确认 | 连续N天未打卡后发送邮件（用户可配置N） |

### 4.3 文案规则

| 场景 | 文案 |
|------|------|
| 待打卡状态 | "还活着吗？" |
| 打卡成功 | "恭喜你又活过了一天！" |
| 昨日未打卡 | "一定是因为昨天过得很开心吧，才忘了打卡" |
| 连续未打卡 | "好久不见，你还好吗？" |
| 里程碑7天 | "连续7天，你是真的在认真活着" |
| 里程碑30天 | "一个月了，生活在继续" |
| 里程碑100天 | "100天，你已经证明了自己" |
| 里程碑365天 | "一整年，这是属于你的史诗" |
